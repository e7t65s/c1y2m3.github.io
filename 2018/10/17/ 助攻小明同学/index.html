一次Jboss反序列化渗透

c1y2m3’blog
2018-10-17

（已修复)

最近闲来无事，正好小明同学发给了我一个jboss站，确认存在 Jboss反序列化漏洞，叫我帮忙搞下，我爽快的答应了，于是打开了神器：

查找到 #User Current Directory: D:\jboss-6.1.0.Final\bin

在寻找jboss web绝对路径中 遇到了点波折，最后找到 根目录网站 ：

故上用冰蝎传了个asp免杀马 ，发现会报500的解析报错，但是解析是没问题的， 连接之 。

os类型:Windows server 2008 查看远程端口是否开启以及端口号 ：

通过16进制转10进制 ：set /a a=0xd3d =3389

原先的思路是，创建个用户 添加到管理组 然后连接到服务器的3389端口，可是没想到 ，无法连接， 应该是管理员把3389禁止外连了，不过没关系，直接把3389反向代理出来。

使用Regeorg 把webshell的服务 与本机的9999端口 建立 socks代理 。

（由于当时没保存完图，后面就不上图了。）
这里要设置下kali下的 /etc/proxychains.conf

去连接3389端口 ==> proxychains rdesktop ip

这里登陆 创建好的用户，用冰蝎 上传 mimikatz 发现打开exe，来了个提示:
文件所在的卷已被外部更改 文件不再有效。经过一波折腾，原来是QQ管家的问题，果断卸载， 可惜最后还是没dump到管理员的密码 ，哦，fuck…..

后面找了个bat脚本去提取内存中的密码，提取出sam，用 saminside 去抓密码 。

Administrator NT-hash 拿去cmd5没破出来， 想想还是不改administrat的密码了。

后面自己又把 端口扫了一下 ，发现了个9090 ZooKeeper 发现….

直接就登陆进来了，(这是什么操作?把密码写在主页上…)。

发现了一些敏感信息：

由于没找到数据库的入口，查看域 ，# net user /domain，本来还想来个内网渗透，正好最近一直在研究内网的思路，可是发现没有在域里面，也就不提权了吧，oh my god…擦屁股走人。

总结：

这是一个失败的渗透测试，不过还是得感谢小明同学。
