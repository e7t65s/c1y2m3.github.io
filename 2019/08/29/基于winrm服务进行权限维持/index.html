    WinRM 是 Microsoft 对 WS-Management 协议的实现,WS-Management 协议即一种基于标准简单对象访问协议[SOAP]的 “防火墙友好” 协议,它让来自不同供应商的硬件和操作系统能够互相操作

    Winrm服务http默认启动监听5985，https端口则5986，系统默认是没有启动服务，
    winrm quickconfig -q 在命令行下开启服务，防火墙会自动添加例外规则。

    默认客户端连接则会提示Winrs error:WinRM 客户端无法处理该请求。如果身份验证方案与 Kerberos 不同，或者客户端计算机…,则需在客户端添加一个TrustedHost表，*代表信任远程任意主机。

    Set-Item WSMan:localhost\client\trustedhosts -value *

    windows内置winrs命令行操作 ,则弹回个半交互cmd shell，

    winrs -r:http://192.168.1.232:5985/wsman -u:administrator -p:XXXX cmd

    Winrm服务底层通讯协议使用的是http,且会在http.sys上注册wsman前缀，HTTP.sys驱动中有个功能，只需要注册各自不同的url前缀即可实现端口共享。如该机器存在web服务端口，则可以利用修改winrm端口来实现权限维持。

    winrm set winrm/config/Listener?Address=*+Transport=HTTP @{Port=”80”}

    可以看到原winrm服务端口5985已关闭，通过web服务端口连接到windows服务器。
